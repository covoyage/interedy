# 2026-04-11

## Interedy Redis 管理工具搭建完成

从零搭建了 Tauri 2.0 + React + TS + TailwindCSS 的 Redis 管理工具：

- 前端 6 个核心组件: Sidebar, KeyTree, KeyDetailPanel, TerminalPanel, AddKeyModal, StatusBar
- Rust 后端 27 个 Tauri command 覆盖连接管理、5 种数据类型 CRUD、TTL、命令执行、DB 信息
- 前后端均编译通过，应用可正常启动运行
- 连接配置持久化到 ~/.config/interedy/connections.json

## 主题切换 + 国际化

- TailwindCSS darkMode: "class" 策略，所有组件适配 dark: 前缀

- LocaleProvider: zh-CN / en-US 双语，localStorage 持久化

- 从右上角浮动按钮移到 Sidebar 左下角，作为底部固定栏
- 左侧主题切换（Sun/Moon + Light/Dark），右侧语言切换（Languages + EN/中文）

## DB 切换功能

- Rust 后端: `switch_db` 命令，通过重建 redis::Client 到新 db 实现切换（比 SELECT 更可靠）
- `connect` 命令返回当前 db 号（u8）
- AppState.clients 类型改为 `HashMap<String, (redis::Client, u8)>` 跟踪当前 db
- 连接配置的 db 作为默认库，连接成功后 Sidebar 显示 db 切换下拉（db0~db15）
- 切换 db 后自动清空 selectedKey 刷新数据
- i18n: sidebar.database 文案

## AddKeyModal 类型选择改为下拉框

- 原来 5 个类型按钮横排（带 icon），改为 `<select>` 下拉选择框
- 移除 FileText/Hash/List/Tag/BarChart3 icon 导入，改用 ChevronDown 下拉箭头
- KEY_TYPES 去掉 icon 字段，select option 只显示 label
- 样式与其他 input 统一：appearance-none + 右侧 ChevronDown 绝对定位
- 原生 select 在 macOS 上 options 弹窗与 select 对不齐，改为自定义下拉组件（button + 绝对定位面板）
- 点击外部关闭下拉（useRef + mousedown listener）

## AddKeyModal 各类型专属表单

- 原来 Hash 用 `field:value` 冒号拼接、ZSet 用 `score:member` 冒号拼接，不直观
- 改为各类型独立表单：
  - String: textarea
  - Hash: field + value 双输入框，支持添加多行（addField）
  - List: 多个 input，支持添加多行（addItem）
  - Set: 多个 member input，支持添加多行（addItem）
  - ZSet: score + member 双输入框，支持添加多行（addItem）
- 每行有删除按钮（Trash2 icon），至少保留一行
- 添加按钮在标签行右侧，Plus icon
- i18n 新增 addField / addItem 文案

## 测试连接功能

- Rust 后端: `test_connection` 命令，接受 host/port/password/db 独立参数，不依赖已保存连接
- 连接后执行 PING，成功返回 "PONG"，失败返回错误信息
- Sidebar 连接表单底部加"测试连接"按钮，左侧显示
- 测试状态: idle → testing(Loader2 动画) → success(绿色 ✓) / fail(红色 ✗ + 错误信息)
- 删除了旧的重复 test_connection 定义（接受 RedisConnection 参数的版本）
- i18n: testConnection / testSuccess / testFail

## 自动更新 + GitHub Actions

参考 kairoa 实现：

### Rust 后端
- Cargo.toml 添加 `tauri-plugin-updater = "2"` 和 `tauri-plugin-process = "2"`
- lib.rs 注册 `.plugin(tauri_plugin_updater::Builder::new().build())` 和 `.plugin(tauri_plugin_process::init())`
- macOS 菜单添加 "Check for Updates..." 菜单项，点击后 emit `check-for-updates` 事件

### 前端
- 添加 `@tauri-apps/plugin-updater` 和 `@tauri-apps/plugin-process` npm 包
- 新增 UpdateDialog 组件：检查更新 → 有更新/已是最新 → 下载进度 → 重启确认
- App.tsx 监听 `check-for-updates` 事件打开 UpdateDialog
- i18n: update.check / checking / available / latest / download / downloading / downloadComplete / restartToUpdate / restartConfirm / restartLater / error / errorHint / releaseNotes

### Tauri 配置
- tauri.conf.json: `bundle.createUpdaterArtifacts: true`, `plugins.updater` 配置 endpoint + pubkey
- capabilities/default.json: 添加 `updater:default` 和 `process:default` 权限

### 签名密钥
- 生成: `CI=true pnpm tauri signer generate -w ~/.tauri/interedy.key`
- 公钥: ~/.tauri/interedy.key.pub (已写入 tauri.conf.json)
- 私钥: ~/.tauri/interedy.key (需配置到 GitHub Secrets: TAURI_SIGNING_PRIVATE_KEY)

### GitHub Actions
- .github/workflows/build.yml: 多平台构建 (macOS x86_64/arm64, Ubuntu, Windows)
- 触发条件: push tag v*, PR, workflow_dispatch
- 构建: 从 tag 提取版本号 → 更新 Cargo.toml/tauri.conf.json 版本 → tauri-action 构建
- 生成 updater JSON: 从 Release assets 匹配平台+签名 → 生成 latest.json → 上传到 Release
- Rust 后端: macOS 原生菜单栏添加 "About Interedy" 菜单项
  - 点击后通过 `window.emit("show-about", ())` 通知前端
  - 同时添加了 Edit 菜单（Undo/Redo/Cut/Copy/Paste/Select All）
- 前端: App.tsx 监听 `show-about` 事件打开 AboutDialog
- i18n: about.subtitle / about.version / about.viewOnGitHub / about.license
